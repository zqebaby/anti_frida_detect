根据打印结果，即可拿到key和msg，key是一个固定串，”F8mrVRBmPDkx5SjTLK2fvdHvxwxc8n2b”，msg由header中的关键参数同请求体中的参数，合并去除掉空值之后，通过&/=的形式连接起来，类似urlencode，但是不需要转译，例：

apiVersion=1.0.1&channelId=AVG10003&clientVer=1.0.04&devId=0010337e0f175194&dev_model=ONEPLUS A6000&lang=en&openid=0010337e0f175194&session=ff23f1f98c0c2b6069af22cdf566123e&sid=0&ts=1722830790&uid=165167403

参考上述流程，可以知道，play_info解密函数decryptStr函数底层就是使用的AES对称加密算法，将base64解密之后，结合密钥之后解密。

根据java层算法，得知解密之后还需要 使用zlib进行一次解压。

import base64
import hashlib
import hmac
import zlib
from Crypto.Cipher import AES 


def decrypt_aes(encrypted_data, aes_key, aes_iv):
    # Base64 decode if the encrypted_data is base64 encoded
    encrypted_data = base64.b64decode(encrypted_data)

    # Initialize cipher with key and iv
    cipher = AES.new(aes_key, AES.MODE_CBC, aes_iv)

    # Decrypt data
    decrypted_data = cipher.decrypt(encrypted_data)

    # Remove padding (Assuming PKCS7 padding)
    padding_length = decrypted_data[-1]
    decrypted_data = decrypted_data[:-padding_length]

    return decrypted_data

def sign(headers, req_data_dict):
    """req sign"""
    fixed_k = "F8mrVRBmPDkx5SjTLK2fvdHvxwxc8n2b"
    d = {
        k: headers[k]
        for k in [
            "uid",
            "channelId",
            "ts",
            "apiVersion",
            "session",
            "lang",
            "devId",
            "clientVer",
        ]
    }
    d.update(req_data_dict)
    l = [(k, v) for k, v in d.items() if v]
    l.sort(key=lambda x: x[0])
    msg = "&".join([f"{k}={v}" for k, v in l])
    return hmac.new(fixed_k.encode(), msg.encode(), hashlib.sha256).hexdigest()

ReelShort 算法逆向概要

  • ReelShort是知名短剧出海应用，通过逆向可获取其请求签名算法
  • API请求中有两个关键加密参数：sign签名参数和play_info的base64加密串
  • 通过jadx-gui和字符串搜索"sign"定位到签名函数位置

关键加密算法

  • sign签名使用的是HMAC-SHA256算法，密钥为固定字符串"F8mrVRBmPDkx5SjTLK2fvdHvxwxc8n2b"
  • 签名内容由header和请求体中的参数合并后按key排序，通过&/=连接形成
  • play_info解密使用AES对称加密算法，解密后还需进行zlib解压
  • 成功实现了Python版本的解密和签名生成函数
